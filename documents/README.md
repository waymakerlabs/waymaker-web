```
documents/
├── README.md              # 이 파일
├── daily-log-*.md         # 일일 작업 로그
├── ARCHITECTURE.md        # 아키텍처 문서 (선택)
├── IMPLEMENTATION_PLAN.md # 구현 계획서 (선택)
└── ...
```

## Daily Log

일일 작업 로그는 다음 형식으로 관리됩니다:

- **파일명**: `daily-log-YYYY-MM-DD.md`
- **목적**: 매일 수행한 작업 기록 및 다음 세션 컨텍스트 유지

### 활용 방법

1. **작업 완료 후**: "데일리로그" 또는 "daily log" 명령으로 로그 작성
2. **다음 세션 시작 시**: 최근 daily log의 "다음 세션을 위한 프롬프트" 섹션 활용
